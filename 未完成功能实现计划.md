## 三、API问题诊断与修复

### 3.1 当前API调用流程
```
用户点击"开始生成"
  → startGeneration()
  → resourceGenerator.generateResources()
  → multiAgentScheduler.executeTask()
  → chatCompletion()  ← 可能失败
  → 返回结果解析
```

### 3.2 MiniMax-M2.7 API 响应格式排查

根据文档，Anthropic兼容API的响应格式：
```json
{
  "id": "xxx",
  "type": "message",
  "role": "assistant",
  "content": [
    {"type": "text", "text": "..."},
    {"type": "thinking", "thinking": "..."}
  ],
  "model": "MiniMax-M2.7",
  "stop_reason": "end_turn"
}
```

**可能问题**：
1. content数组中text block的提取逻辑
2. 需要检查实际API返回

### 3.3 修复计划

```typescript
// api.ts 修复点
1. 添加详细日志输出原始响应
2. 支持多种content block类型
3. 处理可能的thinking block
4. 添加超时和重试逻辑
```
